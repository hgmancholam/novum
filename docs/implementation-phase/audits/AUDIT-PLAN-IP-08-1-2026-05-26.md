## 4. Blind-Path Findings

### F-01 (minor) — `evaluate_with_judge` test recomputation depends on full fixture state
- **Location:** plan §6.4, `test_agent_tasks_draft.py` update.
- **Type:** untestable acceptance criterion (risk only).
- **Affected RF:** RF-12.
- **Severity:** minor.
- **Description:** The existing test for `evaluate_with_judge` previously asserted `structural_confidence == state.coverage_ratio()`. After the swap, the asserted value becomes the full weighted S, which depends on `evidence`, `contradictions`, and `covered_claims` being meaningfully populated in the test fixture. If the current fixture has empty evidence, the new score will be `0.35 · coverage` (because agreement/diversity/no_conflict collapse to 0 or 1) — still deterministic, but the test author must recompute by hand.
- **Fix recommendation:** No plan change required. Coder must follow §6.4 verbatim and verify with `pytest -v`. Optionally, the plan could explicitly note "if the fixture has no evidence, expected score = `0.35 * coverage + 0.15 * 1.0`" but this is a Coder-side calculation, not a planning gap.
