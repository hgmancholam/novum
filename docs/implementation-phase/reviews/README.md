# Code Reviews

This folder contains code review reports generated by the Reviewer Agent.

## Structure

- Each review follows the naming convention: `REVIEW-{US-number}-{iteration}-{date}.md`
- Reviews include scores and feedback
- Maximum 5 iterations per implementation

## Scoring

- **Score ≥ 9/10**: Implementation approved
- **Score < 9/10**: Requires revision (feedback provided)
- **Max iterations**: 5 (escalate to manual review if not resolved)

## Review Criteria

1. Code quality and best practices
2. Test coverage
3. Architecture compliance
4. Documentation
5. Security considerations
